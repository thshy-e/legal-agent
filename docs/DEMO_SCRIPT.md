# 面试演示脚本

这份脚本用于 5-8 分钟内展示项目价值。建议先启动服务，再按顺序输入问题。

## 启动

```powershell
.\venv\Scripts\python.exe start_server.py
```

打开 <http://localhost:8000>。

## 演示主线

### 1. 确定性赔偿计算

输入：

```text
月薪10000，工作3年，被公司突然辞退且无补偿，能拿多少？
```

观察点：

- 路由为 `qa`。
- 结构化面板展示赔偿金额 `60,000`。
- 计算公式为 `3 x 10000 x 2`，赔偿类型为 `2N赔偿`。

### 2. 普通法条问答不被污染

继续同一会话输入：

```text
劳动法第68条是什么？
```

观察点：

- 仍为普通 `qa`。
- 不应再出现上一轮 `60000`、`2N`、`计算公式` 或旧案件时间线。
- 说明系统具备上下文隔离策略，不是简单拼接历史。

### 3. 案件连续追问

继续输入：

```text
结合我上面的情况，劳动合同法第47条怎么适用？
```

观察点：

- 允许复用前文工资、年限、解除事实和计算结果。
- 可解释第47条经济补偿规则如何作为 2N 的基础。

### 4. 案件预判

继续输入：

```text
那我这个案子下一步怎么办，胜诉概率怎么样？
```

观察点：

- 路由到 `judge`。
- 回答应围绕证据、公司解除理由、仲裁请求和维权路径展开。

### 5. 文书生成

继续输入：

```text
用以上内容生成劳动仲裁申请书
```

观察点：

- 路由到 `doc`。
- 文书应包含请求事项、事实理由、证据清单、落款。
- 能引用前文赔偿金额或计算摘要。

### 6. 风险评估

继续输入：

```text
公司这样处理有什么法律风险？
```

观察点：

- 路由到 `risk`。
- 回答从公司合规风险和劳动者维权角度展开。

## 可讲述的技术点

- `session_id` 是会话边界，新会话不会继承旧案件事实。
- UI 模式只是偏好，后端路由仲裁会优先识别明确计算/文书意图。
- 赔偿计算优先走确定性公式引擎，避免 LLM 算错。
- 普通法条问答默认不注入案件上下文，避免多轮污染。
- 测试用例覆盖同一会话跨功能混跑。
