### 2.3 失败用例分析

#### 问题1：N+1代通知金未实现（CAL-02）
**现象**：合法无过失辞退时，系统只计算N倍补偿，未包含+1代通知金

**原因**：当前代码只区分"非法辞退(2N)"和"合法解除(N)"，未实现N+1场景

**修复建议**：
```python
# 在 reason 参数中增加 "n_plus_1" 选项
if reason == "非法辞退":
    multiplier = 2
elif reason == "n_plus_1":
    multiplier = 1  # N部分
    # 额外加1个月代通知金
    total = months * base_salary + base_salary
else:
    multiplier = 1
```

#### 问题2：12年封顶逻辑缺失（CAL-04）
**现象**：高薪员工（社平3倍以上）工作超过12年时，未按12年封顶计算

**原因**：代码只实现了工资基数封顶（社平3倍），未实现年限封顶（最高12年）

**修复建议**：
```python
# 高薪封顶：年限最高12年
if salary > avg_salary_local * 3:
    months = min(months, 12)
```

#### 问题3：工龄折算逻辑错误（CAL-11、CAL-18）
**现象**：
- 6.2年 → 实际折算6个月，预期6.5个月
- 8.3年 → 实际折算8个月，预期8.5个月

**原因**：当前逻辑 `int(years) + (1 if years % 1 >= 0.5 else 0)` 只在小数部分≥0.5时进1，但测试用例期望任何有零头的年份都按半年算

**修复建议**：
```python
# 修改为：有零头就加0.5年
if years < 0.5:
    months = 0.5
else:
    months = int(years) + 0.5 if years % 1 > 0 else int(years)
```

#### 问题4：最低工资兜底缺失（CAL-19）
**现象**：员工月薪低于当地最低工资时，未按最低工资计算

**原因**：代码未实现最低工资兜底逻辑

**修复建议**：
```python
# 增加最低工资参数
min_salary = context.get("min_salary", 0)
base_salary = max(min_salary, min(salary, cap))
```

### 3.4 评分标准

| 评估维度 | 权重 | 评分规则（1-5分制） |
|----------|------|---------------------|
| 答案正确性 | 40% | 5分：完全正确；4分：基本正确；3分：部分正确；2分：错误较多；1分：完全错误 |
| 法条引用准确性 | 30% | 5分：精准引用；4分：正确但不完整；3分：基本正确；2分：引用错误；1分：未引用 |
| 响应时间 | 10% | 5分：≤1秒；4分：1-3秒；3分：3-5秒；2分：5-10秒；1分：＞10秒 |
| 回复友好度 | 20% | 5分：通俗易懂、有实操指引；4分：较通俗；3分：一般；2分：晦涩；1分：无法理解 |

### 3.5 综合评分计算

```
综合评分 =（答案正确性×40% + 法条引用×30% + 响应时间×10% + 友好度×20%）× 20
```

将1-5分制换算为100分制。

## 四、测试结论与建议

### 4.1 赔偿计算引擎（通过率75%）

**优势**：
- 基础N和2N计算准确
- 工资封顶逻辑（社平3倍）实现正确
- 半年工龄折算基本正确

**需改进**：
1. 实现N+1代通知金计算
2. 实现12年封顶逻辑
3. 修复工龄折算bug（有零头年份应按半年算）
4. 增加最低工资兜底逻辑
5. 增加年休假、加班费等专项计算

### 4.2 对比实验测试（成功率100%，关键词命中率18%）

**优势**：
- 系统稳定性好，50题全部成功响应
- 问答类能正确引用法条（命中率42.6%）
- 响应速度可接受（问答32秒，其他4-7秒）

**需改进**：
1. **计算类问题**：系统未自动调用计算工具，建议优化RouterAgent的意图识别
2. **关键词命中率低**：回复内容与预期关键词不完全匹配，需优化Prompt模板
3. **响应时间**：问答类平均32秒偏长，可考虑缓存或异步处理

### 4.3 改进优先级

| 优先级 | 改进项 | 预期效果 |
|--------|--------|----------|
| P0 | 修复赔偿计算引擎bug | 通过率提升至90%+ |
| P0 | 优化RouterAgent计算类意图识别 | 计算类关键词命中率提升 |
| P1 | 优化Prompt模板 | 提高回复质量和关键词命中率 |
| P2 | 增加响应缓存 | 降低平均响应时间 |
