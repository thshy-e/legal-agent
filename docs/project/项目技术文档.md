> 维护说明：本文件保留项目技术说明和比赛材料上下文；最新的启动方式、目录结构和 API 契约以仓库根目录 `README.md`、`docs/ARCHITECTURE.md`、`docs/API.md` 为准。

## 一、项目概述

**项目名称**：劳法智枢（Legal AI Agent）

**核心目标**：构建一个专业的劳动法领域AI决策系统，通过多智能体协作，为劳动者提供法律咨询、赔偿计算、案件预判和文书生成等一站式法律服务。

**系统特点**：
- 劳动争议场景专用，针对劳动法领域深度优化
- 多智能体协作架构，分工明确
- 内置知识库（法条+案例），支持RAG检索增强
- 赔偿计算引擎，精准计算各类经济补偿
- 前后端分离，支持Web端访问

## 三、核心功能模块

### 3.1 多智能体路由系统（RouterAgent）
**功能**：智能识别用户意图，将请求分发到对应的专业智能体

**技术实现**：
- 使用PydanticOutputParser确保输出格式稳定
- 支持4种意图分类：qa（问答）、doc（文书）、risk（风险）、judge（预判）
- 内置兜底机制，解析失败时默认走qa流程

**路由意图**：
| 意图 | 说明 | 对应Agent |
|------|------|-----------|
| qa | 法律知识问答 | QAAgent |
| doc | 法律文书生成 | DocAgent |
| risk | 企业政策风险评估 | RiskAgent |
| judge | 案件胜诉预判 | JudgeAgent |

### 3.2 法律问答智能体（QAAgent）
**功能**：回答劳动法相关问题，引用法条和案例

**技术实现**：
- 基于LangGraph的create_react_agent构建
- 支持工具调用：search_law（检索法条）、search_case（检索案例）、calculate_severance_pay（计算赔偿）
- 内置MemorySaver支持多轮对话记忆
- 使用ReAct模式（推理-行动-观察）

### 3.3 文书生成智能体（DocAgent）
**功能**：根据用户描述生成劳动仲裁申请书等法律文书

**技术实现**：
- 直接调用LLM生成文书
- 结构化Prompt模板确保格式规范

### 3.4 风险评估智能体（RiskAgent）
**功能**：评估企业政策或做法的法律风险

**输出格式**：
- 风险等级（高风险/中风险/低风险/合规）
- 涉及法条及解释
- 可能的法律后果
- 劳动者应对建议

### 3.5 案件预判智能体（JudgeAgent）
**功能**：预判劳动仲裁案件的胜诉概率

**输出格式**：
- 案件性质认定
- 胜诉概率评估（百分比+理由）
- 关键证据建议
- 可能赔偿范围

### 3.6 赔偿计算引擎（calculate_severance_pay）
**功能**：精准计算各类劳动法场景下的经济补偿

**支持场景**：
- 经济补偿金（N）
- 违法解除赔偿金（2N）
- 代通知金（N+1）
- 未签劳动合同双倍工资
- 高薪封顶计算（社平工资3倍）
- 工龄折算（不足半年按0.5年，超半年按1年）

**计算逻辑**：
```python
# 工资封顶判断
cap = avg_salary_local * 3
base_salary = min(salary, cap)

# 工作年限折算
if years < 0.5:
    months = 0.5
else:
    months = int(years) + (1 if years % 1 >= 0.5 else 0)

# 非法辞退赔偿金为2N，合法为N
multiplier = 2 if reason == "非法辞退" else 1
total = months * base_salary * multiplier
```

### 3.7 知识库管理系统（build_db.py）
**功能**：管理法条和案例的向量知识库

**支持操作**：
- 从docx文件加载法条和案例
- 构建向量库（ChromaDB）
- 增量添加文档
- 按ID删除文档
- 相似度检索

**数据源**：
- 劳动法.docx：劳动合同法、劳动法等法条
- 劳动法经典案例.docx：典型劳动争议案例

## 六、API接口说明

### 6.1 聊天接口
**端点**：`POST /api/chat`

**请求体**：
```json
{
    "query": "用户问题",
    "mode": "qa|doc|risk|judge"  // 可选，强制指定Agent模式
}
```

**响应体**：
```json
{
    "reply": "AI回复内容（Markdown格式）",
    "route": "qa|doc|risk|judge"  // 实际使用的Agent
}
```

- Python 3.9+

1. **PydanticOutputParser**：彻底解决LLM输出格式不稳定问题

3. **RAG检索增强**：基于知识库的回答，避免幻觉
4. **赔偿计算引擎**：支持复杂场景的精准计算
5. **工具调用（Tool Calling）**：LangGraph原生支持，提升Agent能力
6. **多轮对话记忆**：基于thread_id的会话隔离
